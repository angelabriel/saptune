November 10, 2025 | By: Sören Schmidt

Last week the maintenance update saptune 3.2.1 has been released for:
  • SUSE Linux Enterprise Server for SAP Applications 15 SP4
  • SUSE Linux Enterprise Server for SAP Applications 15 SP5
  • SUSE Linux Enterprise Server for SAP Applications 15 SP6
  • SUSE Linux Enterprise Server for SAP Applications 15 SP7
In a few days:
  • SUSE Linux Enterprise Server for SAP Applications 12 SP5
  • SUSE Linux Enterprise Server for SAP Applications 15 SP3
also will follow.

The update brings primarily a fix and a SAP Note update for cloud environments:

Restore CPU performance settings on AWS and Google Cloud

With 3.2 we limited setting some CPU tunings (energy_perf_bias, governor, force_latency) to Intel bare-metal only overlooking that on AWS and GCP some instance types indeed allow CPU tuning. 3.2.1 is fixing this and set those tunings on AWS and GCP again.

Update SAP Note 2993054

The recommendation to set net.ipv4.tcp_retries on Application server HA clusters was added to SAP Note "2993054 - Recommended settings for SAP systems on Linux running in Azure virtual machines" in version 4. After consulting with the Azure team at SAP saptune will set this recommendation in general as saptune does not yet detect HA clusters and their type.

That's it.

You might have noticed, that not all current SAP Note recommendations are part of 3.2.1 like the minimal sssd package, the TSX setting or the C-state recommendation for AWS. These recommendations require additional coding (support for optional packages, a kernel version tag, state name support for force_latency) which will take a bit more time. We did not want to delay 3.2.1 further, so expect the missing tunings with 3.2.2 around January/February 2026. {% endblock content %}
