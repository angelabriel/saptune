February 13, 2026 | By: Sören Schmidt

Yesterday the maintenance update saptune 3.2.2 has been released for:
  • SUSE Linux Enterprise Server for SAP applications 15 SP4
  • SUSE Linux Enterprise Server for SAP applications 15 SP5
  • SUSE Linux Enterprise Server for SAP applications 15 SP6
  • SUSE Linux Enterprise Server for SAP applications 15 SP7
It contains only a HotFix for the problem described in the TID https://support.scc.suse.com/s/kb/SAP-HANA-swap-and-system-hangs-with-THP-on-multi-NUMA-node-systems[SAP HANA swap and system hangs with THP on multi-NUMA node systems].

You will also find references to that TID in the SAP Notes:
  • 2684254 - SAP HANA DB: Recommended OS settings for SLES 15 / SLES for SAP Applications 15
  • 2131662 - Transparent Huge Pages (THP) on SAP HANA Servers
The 3.2.2 release contains an additional Note HotFix000022286 which is part of the Solutions HANA, NETWEAVER+HANA, S4HANA-APP+DB and S4HANA-DBSERVER.

It will set on SLES for SAP applications 15 SP5-7 Intel x86_64 systems the parameters kernel.mm.transparent_hugepage.defrag and kernel.mm.transparent_hugepage.khugepaged.defrag as described in the TID.

What is a HotFix?

A HotFix is a new way to react fast to critical bugs, that can harm SAP installations.
It is meant to bridge the time with a temporary mitigation until the fix is available
. If this is the case, the next saptune version will contain an update to the HotFix to be applied only when necessary.
One year after the fix is available, we will remove the HotFix from saptune.

We hope to provide an comfortable way to address such issues.

A HotFix release will contain only configuration changes and no new code.

How To Apply this HotFix?

I have one of the Solutions HANA, NETWEAVER+HANA, S4HANA-APP+DB or S4HANA-DBSERVER`applied.

Update to saptune 3.2.2, reboot or just restart the saptune service. If you restart the service, keep in mind that there is no tuning for a short period of time! SAP HANA should be stopped. + With HotFix000022286 being part of those Solutions, it gets applied automatically.

If you cannot afford interruption of SAP HANA, simply execute saptune note apply HotFix000022286.

Finally run saptune verified applied to check if HotFix000022286 is listed and the tuning is compliant.

I have no Solutions applied, just individual Notes including 2684254 setting THP.

Simply execute saptune note apply HotFix000022286 and run a saptune verified applied to check if HotFix000022286 is listed and the tuning is compliant.

What happens next?

As soon as a kernel fix is available, the next saptune version will contain HotFix000022286 with a condition (tag) to be applied on prior kernels, so the temporary mitigation is only set if needed. The HotFix will be removed from `saptune` one year after the kernel fix is available, giving customers enough time to update their systems. {% endblock content %}
