saptune 3.1.3 - Minor Fixes and a Note Update

August 13, 2024 | By: Sören Schmidt

If you check the repositories of SLES for SAP Applications 12 SP5 and 15 SP2-6 you find a `saptune` package 3.1.3.
What does this update contain?

Removal of SAP Note 1868829 from S4HANA-APPSERVER Solution

Thanks to the customer who spotted this mistake, we missed for some time.
SAP Note "1868829 - Startup Issues Because Number of Active I/O Requests to Queue Exceeds aio-max-nr Limit" is actually a Note for SAP HANA and ended up in the S4HANA-APPSERVER Solution by accident. We have removed it from the Solution.
After an update to saptune 3.1.3 the Note will still be applied even if they is not listed anymore as part of the Solution, but after a restart of saptune.service the tuning will be refreshed.

Updates to SAP Note 2578899

The update also contains the latest changes to SAP Note "2578899 - SUSE Linux Enterprise Server 15: Installation Note", meaning starting the sysctl-logger service as well setting kernel.pid_max.

Smaller columns - Better readability

Again a hint from a customer.
On large machines (e.g. large number of CPUs) the table columns of a verify output could become unreadable wide due to the amount of data. Now a column gets wrapped after 32 characters. Hopefully a good compromise.
This one must have bugged people for quite some time, I assume. If you find such nuisances, please let us know!

Bug in CPU detection

On rare occasions CPU tuning features were discovered, where they should not. This has been fixed.

Have a nice day!
